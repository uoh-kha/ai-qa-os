# Purpose

It describes the lifecycle of engineering work from an initial request to continuous improvement.

# Workflow Stages

## 1. Task Definition

Owner:

Human QA Lead

Output:

- Initial task

---

## 2. Product Definition

Owner:

Product Owner

Output:

- Requirements
- User Stories
- Acceptance Criteria

---

## 3. Requirement Analysis

Owner:

Business Analyst

Output:

- Questions
- Clarifications
- Risk Analysis

---

## 4. Test Design

Owners:

Manual QA

Test Architect

Outputs:

- Test Strategy
- Test Cases
- Coverage Analysis

---

## 5. Automation

Owner:

Automation QA

Outputs:

- Playwright Tests
- API Tests
- Test Utilities
- Framework Updates

---

## 6. Technical Review

Owner:

Code Reviewer

Outputs:

- Review Findings
- Improvement Suggestions

---

## 7. CI/CD Integration

Owner:

Owner:

## 9. Reporting

Owner:

## 10. Human Review

Owner:

## 11. Retrospective

Participants:

Human QA Lead

Relevant AI Roles

Outputs:

- Process Improvements
- Role Improvements
- Constitution Updates
- Workflow Improvements

---

# Workflow Principles

Every stage:

- has a single owner;
- receives defined inputs;
- produces defined outputs;
- can request clarification;
- can return work for correction;
- should not bypass previous stages without approval.

---

# Continuous Improvement

The workflow is expected to evolve.

Changes should be based on measurable improvements rather than assumptions.
